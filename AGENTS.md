# FMCodex Agent Guide

本项目是一个 UE5 C++ 足球卡牌双人联网对战游戏。当前阶段以产品规则、技术边界、数据结构和测试思路整理为主。

## 当前工作边界

- 不修改 UE 自动生成的 C++ 文件。
- 不创建蓝图。
- 不实现玩法逻辑。
- 不接入 Steam、EOS 或其他线上服务。
- 不做大规模重构。
- 优先创建和维护文档、目录、命名约定和决策记录。

## 项目目标

做一个双方玩家各自使用足球球员卡、战术卡或事件卡进行回合制或阶段制对战的联网游戏。游戏应先形成稳定、可解释、可测试的规则，再进入代码实现。

## 协作原则

- 先文档，后实现。
- 先最小可玩规则，后扩展内容。
- 规则、网络同步、数据结构和测试用例必须互相对应。
- 新增系统前，先查看 `Docs/08_Decision_Log.md` 是否已有相关决定。
- 如果规则存在歧义，先补充 `Docs/01_Rules_Canonical.md`，不要直接写代码猜测。

## 推荐阅读顺序

1. `Docs/00_Product_Vision.md`
2. `Docs/01_Rules_Canonical.md`
3. `Docs/02_Rules_Glossary.md`
4. `Docs/03_Tech_Architecture.md`
5. `Docs/04_Networking_Model.md`
6. `Docs/05_Data_Schema.md`
7. `Docs/07_Test_Cases.md`
8. `Docs/08_Decision_Log.md`

## 面向新手的约定

- 文档里尽量使用明确、普通的语言。
- 技术文档可以出现专业词，但必须解释它对游戏有什么影响。
- 每次变更都应能回答三个问题：
  - 这个变更解决什么问题？
  - 玩家会感受到什么变化？
  - 以后测试时怎么确认它没坏？
