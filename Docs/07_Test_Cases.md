# 07 Test Cases

本文档记录测试用例草案。当前只写测试思路，不写测试代码。

## 测试目标

未来每条核心规则都应该能被测试验证。测试不是只为了程序员，也是为了让规则变更不会悄悄破坏游戏。

## 规则测试

待补充用例：

- 初始对局能正确创建。
- 双方初始手牌数量正确。
- 当前阶段只能执行允许的操作。
- 不满足费用时不能使用卡牌。
- 使用卡牌后资源正确扣除。
- 卡牌进入正确区域。
- 得分后比分正确变化。
- 比赛结束时胜负正确。

## 联网测试

待补充用例：

- 客户端请求非法操作时，服务器拒绝。
- 服务器结算后双方客户端状态一致。
- 私密手牌不会同步给对手。
- 随机结果在双方客户端显示一致。
- 掉线或超时行为符合规则。

## 数据测试

待补充用例：

- 每张卡都有唯一 CardId。
- 卡牌类型必须是合法类型。
- 卡牌费用不能为负数。
- 描述不能为空。
- Timing 必须匹配规则允许的时机。
- EffectId 必须能被未来规则系统识别。

## 新手可读验收清单

每次以后新增玩法，都应能回答：

- 玩家什么时候能做这个操作？
- 什么情况下不能做？
- 成功后状态怎么变化？
- 失败或被拒绝时玩家看到什么？
- 联网时谁负责判断结果？

## 当前不做

- 不写自动化测试代码。
- 不创建测试地图。
- 不创建测试蓝图。
- 不接真实服务器。
